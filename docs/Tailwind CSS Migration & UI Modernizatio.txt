Tailwind CSS Migration & UI Modernization Prompt

You are a Senior Frontend Architect, Senior React Engineer, and Senior UI/UX Designer.

Your task is to modernize the frontend of the SOC 5 Outbound Truck Request Portal while preserving all existing functionality and business logic.

Project Context

The application is an internal enterprise logistics system used for:

- Truck Requests
- Approval Workflow
- Truck Assignment
- Dock Assignment
- Dispatch Monitoring
- Realtime Status Updates
- Audit Trail
- User Management
- Dashboard & Reports

Current Stack

- React 19
- TypeScript
- Vite
- TanStack Query
- Zustand
- Laravel 12 Backend API
- Supabase PostgreSQL
- Supabase Auth
- Supabase Realtime

Current Styling

- Sass (SCSS)

Target Styling

- Tailwind CSS (latest stable version)

---

Primary Goal

Replace the entire Sass-based styling system with Tailwind CSS while significantly improving the application's UI/UX.

The migration must not change any application behavior, API contracts, routing, authentication, business logic, state management, or data flow.

Only improve the presentation layer.

---

Migration Requirements

Completely remove the dependency on Sass.

Remove:

- SCSS files
- Sass variables
- Mixins
- Nested styles
- Unused CSS
- Legacy global styling

Replace them using:

- Tailwind utility classes
- Tailwind theme configuration
- CSS variables only when necessary
- Minimal custom CSS
